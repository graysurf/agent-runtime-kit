# docs/discussions

Captured discussion / implementation-readiness specs that are **not** (yet) an
executed-and-archived plan bundle — the default output of the
`discussion-to-implementation-doc` skill.

- One dated file per capture: `docs/discussions/<YYYY-MM-DD>-<slug>.md`.
- Coordination material: cleanup-eligible once the work it describes ships or is
  abandoned. Promote to canon (the owning domain docs, or `docs/source/` for
  repo-wide architecture / specs / policy) only if it proves authoritative.
- Not scanned by `plan-tooling` / `plan-archive`. For work that will be executed
  and archived as a plan, use a `docs/plans/<YYYY-MM-DD>-<slug>/` bundle instead.

See `docs/source/docs-placement-retention-policy-v1.md` (placement + retention)
and `core/policies/work-tier-levels.md` (how the tier decides the home).
